概述是您的用户了解您的项目的地方。根据文档集的大小，您可以有一个单独的概述页面（如此页面）或将您的概述内容放在文档登录页面中（如 Docsy 用户指南中）。

尝试在此页面为您的用户回答这些问题：

## 它是什么？

介绍您的项目，包括它做什么或让您做什么，为什么您会使用它，以及它的主要目标（以及它如何实现）。这应该类似于您的 README 描述，尽管如果您愿意，可以在这里更详细一些。

## 为什么我想要它？

帮助您的用户了解您的项目是否会帮助他们。有用的信息可以包括：

- **它适合什么？**：您的项目解决什么类型的问题？使用它有什么好处？

- **它不适合什么？**：例如，指出可能直观地看起来适合您的项目，但由于某种原因不适合的情况。还要提及已知的限制、扩展问题或任何其他可能让您的用户知道项目不适合他们的内容。

- **它还不适合什么？**：突出显示即将推出的任何有用功能。

## 我应该去哪里？

从概述中为您的用户提供下一步。例如：

- [入门](/zh/docs/getting-started/)：开始使用 $project
- [示例](/zh/docs/examples/)：查看一些示例代码！
